## 4. 函数

### 4.1 函数的返回值

> **✅ 规则：**
>
> - 函数输出优先使用返回值，避免通过输出参数来输出.
>
> - 返回值类型：
>   - 优先按值返回
>   - 当拷贝不合适，并且“不存在对象”不是合法状态时，返回 `T&` / `const T&`
>   - 返回已有对象且“可能为空”：返回 `T*` / `const T*`
>
> - 可能失败的返回值：
>
>   - `std::optional<T>` 
>
>   - `std::expected<T, E>`   

C++ 函数的输出应优先通过返回值表达.

默认：按值返回 `T`，不要返回 `const T`（无意义）.



当拷贝不合适，并且“不存在对象”不是合法状态时，返回 `T&` / `const T&`.



返回已有对象且“可能为空”：返回 `T*` / `const T*`.



与对象“可能为空”相似的，还有中途出现了错误，而导致失败.

对于可能失败的返回值：

- 只关心有没有结果：`std::optional<T>` 

- 需要携带错误信息：`std::expected<T, E>`

### 4.2 函数的输入参数

> **✅ 规则：**
>
> **非可选输入参数**
>
> - 小型、廉价拷贝类型：按值传递 `T`
> - 大型或非平凡拷贝类型：按 `const T&`
> - 函数需要无条件保留一份副本：优先按值传递 `T`，函数内 `std::move`
> - 只读字符串且不保存：优先 `std::string_view`
> - 只读连续序列且不保存：优先 `std::span<const T>`
>
> **可选输入参数**
>
> - 可选的小对象输入用 `std::optional<T>`
> - 可选的大对象输入用 `const T*`

**非可选输入参数：**

小型、廉价拷贝类型：按值传递 `T`

大型或非平凡拷贝类型：按 `const T&`

函数需要无条件保留一份副本：优先按值传递 `T`，函数内 `std::move`

```cpp
class Widget {
public:
	explicit Widget(std::string name) : name_(std::move(name)) {}
private:
	std::string name_;
};
```

这里的无条件保留是指不管参数内容是什么、不管分支怎么走，最终一定需要一份副本.

这里再举两个例子：

```cpp
void MaybeSetName(std::string name) {  
  if (IsValid(name)) {
    name_ = std::move(name);			// 不好：不是无条件保留：只有部分路径会保存
  }										
}
```

可以改为：

```cpp
void MaybeSetName(std::string_view name) {
  if (IsValid(name)) {
    name_ = std::string{name};			
  }
}
```

我们仅推荐在无条件保留使用按值传递 `T`，函数内 `std::move`.



只读字符串且不保存：优先 `std::string_view`

只读连续序列且不保存：优先 `std::span<const T>`



**可选输入参数：**

调用函数时，这个输入参数可以“有”，也可以“没有”.

小对象：用 `std::optional<T>`

```cpp
SetTimeout(1000);          // 有 timeout
SetTimeout(std::nullopt);  // 没有 timeout
```

大对象：用 `const T*`

```cpp
Config config{};
LoadConfig(&config);   		// 有 config
LoadConfig(nullptr);  		// 没有 config
```

### 4.3 输出参数 / 输入输出参数

> **✅ 规则：**
>
> - 非可选的输出参数和输入/输出参数通常应使用**引用**.
> - 可选输出参数和可选输入/输出参数通常使用**非 `const` 指针**.
> - 参数顺序：输入参数在前，输出参数和输入/输出参数在后

有时函数输出也可以通过输出参数（output parameters）或输入/输出参数（in/out parameters）提供.

其中，输出参数是指函数不关心它调用前是什么值，只会把结果写进去的参数.

```cpp
bool Foo(std::string_view text, Size& size);
Size size{};  // 旧值无意义
if (Foo("zoo", size)) {
  // size 被函数写成解析结果
}
```

而输入/输出参数：先读旧值，再写新值

```cpp
void NormalizePath(std::filesystem::path& path);
std::filesystem::path path {"./foo/../bar.txt"};

NormalizePath(path); // path 变成 "bar.txt" 或规范化后的路径
```

**优先使用返回值**，而不是输出参数或者输入/输出参数.

只有在返回值不适合表达时，才使用输出参数或输入/输出参数.



非可选的输出参数和输入/输出参数通常应使用引用，因为引用不能为空.

使用非 `const` 指针表示可选输出参数和可选输入/输出参数.'



参数顺序：输入参数在前，输出参数和输入/输出参数在后.

### 4.4 编写简短函数

> **✅ 规则：**
>
> - 优先编写小而专注的函数.

我们承认长函数有时是合理的, 因此并不硬性限制函数的长度. 

如果函数超过 40 行, 可以思索一下能不能在不影响程序结构的前提下对其进行分割.



即使一个长函数现在工作的非常好, 一旦有人对其修改, 有可能出现新的问题, 甚至导致难以发现的 bug. 

使函数尽量简短, 以便于他人阅读和修改代码.



在处理代码时, 你可能会发现复杂的长函数. 

不要害怕修改现有代码: 如果证实这些代码使用 / 调试起来很困难, 或者你只需要使用其中的一小段代码, 考虑将其分割为更加简短并易于管理的若干函数.

### 4.5 函数重载

> **✅ 规则：**
>
> - 当各变体在语义上没有差异时，可以对函数进行重载.

若要使用函数重载, 则必须能让读者一看调用点就胸有成竹, 而不用花心思猜测调用的重载函数到底是哪一种. 这一规则也适用于构造函数.



你可以编写一个接受 `const std::string&` 的函数，并用另一个接受 `const char*` 的函数对其重载（然而，在这种情况下请考虑使用 `std::string_view `代替）

```cpp
class MyClass {
 public:
  void Analyze(const std::string& text);
  void Analyze(const char* text, size_t textlen);
};
```

这是一个好的重载：语义完全一致.



而可以接受的重载：参数数量不同，但仍是同一语义.

```cpp
class Logger {
 public:
  void Log(std::string_view message);
  void Log(std::string_view message, int level);
};
```

而不好的重载是：名字一样，但语义不同.

### 4.6 缺省参数(默认参数)

> **✅ 规则：**
>
> - 当默认值可保证始终相同时，允许在非虚函数上使用缺省参数.

对于虚函数, 不允许使用缺省参数, 因为在虚函数中缺省参数不一定能正常工作. 如果在每个调用点缺省参数的值都有可能不同, 在这种情况下缺省函数也不允许使用. (例如, 不要写像 `void f(int n = counter++);` 这样的代码.)

### 4.7 函数返回类型后置语法

> **✅ 规则：**
>
> - 只有在常规写法 (返回类型前置) 不便于书写或不便于阅读时使用返回类型后置语法.
