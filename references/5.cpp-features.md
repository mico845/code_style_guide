## 5. C++特性

### 5.1 类型

**整型**

在内置的 C++ 整数类型中，唯一使用的是 `int`.

如果程序需要不同大小的整数类型，请使用来自 `<cstdint>` 的定宽整数类型，例如 `std::int16_t`.

如果你的值可能会大于或等于 2^31，请使用 64 位类型，例如 `std::int64_t`.

请记住，即使你的值本身不会超出 `int` 的范围，它在中间计算中也可能被用于需要更大类型的运算.

遇到不确定时，选择更大的类型.



**浮点类型**

在内置的 C++ 浮点类型中，唯一使用的是 `float` 和 `double` 。你可以假定这些类型分别表示 IEEE-754 的 binary32 和 binary64.

不要使用 `long double` ，因为它会产生不可移植的结果.



**`nullptr`**

对指针使用 `nullptr` ，对字符使用 `'\0'` （而不是 `0` 字面量）.



`sizeof`

优先使用 `sizeof(varname)` 而不是 `sizeof(type)`.



**架构可移植性**

编写与架构无关的可移植代码.不要依赖仅在某一处理器上存在的 CPU 特性.

- 打印数值时，使用类型安全的数字格式化库，例如 [`absl::StrCat`](https://github.com/abseil/abseil-cpp/blob/master/absl/strings/str_cat.h), [`absl::Substitute`](https://github.com/abseil/abseil-cpp/blob/master/absl/strings/substitute.h), [`absl::StrFormat`](https://github.com/abseil/abseil-cpp/blob/master/absl/strings/str_format.h), 或者 [`std::ostream`](https://google.github.io/styleguide/cppguide.html#Streams)  ，而不是 `printf` 系列函数.

- 如果需要将内存地址作为整数处理，请将它们存储在 `uintptr_t` 中，而不是 `uint32_t` 或 `uint64_t` 中.

- 根据需要使用花括号初始化来创建 64 位常量.例如：

  ```cpp
  int64_t my_value{0x123456789};
  uint64_t my_mask{uint64_t{3} << 48};
  ```

- 使用可移植的浮点类型；避免使用 `long double`.

- 使用可移植的整型；避免使用 `short` 、 `long` 和 `long long`.

---

### 5.2 宏

避免任何宏常量，应使用 `constexpr`.

宏仅用于 CMake 配置工程.
